# Experiment data

This directory contains input data.

## Synthetic inputs

- `synthetic/worlds/`: the four SDF worlds `01_sparse.world`, `02_dense.world`, `03_maze_wide.world`, and `04_maze_narrow.world`.
- `synthetic/occupancy_grids/`: 0.05 m Nav2 map YAML/PGM pairs for each Experiment 2 environment and coverage condition.

The occupancy grids are generated by the Experiment 1 multi-resolution map pipeline.
For each requested coverage, the pipeline records the first depth-wide quadtree frontier whose confirmed-free area reaches the target, treats unresolved mixed cells as occupied, and exports the result as a wall-inclusive Nav2 map.
Although generated, these checked-in maps are frozen inputs for Experiment 2.

Coverage directory names identify requested thresholds rather than exact achieved ratios.
Regenerate them with the corresponding config under `configs/multi_resolution_map/`; see the [Experiment 1 multi-resolution map procedure](../experiments/exp1/README.md#multi-resolution-map-and-nav2-grid-generation).

Experiment 2 start-goal pairs are stored with the condition results at `results/exp2_planner_sim/<environment>/coverage_<percent>/start_goal_pairs.csv`.
This ensures that all PPM and Nav2 planners use the same pairs.

## Real-environment inputs

- `real/occupancy_grids/ku_outdoor.yaml` and `ku_outdoor.pgm`: the Nav2 map used by Experiment 3.
- `real/start_goal_pairs/ku_outdoor.csv`: the two fixed start-goal pairs shared by every Experiment 3 planner.
- `real/ppm/`: the PPM data for `1of1`, `1of3`, and `adjusted`.
