# PPM A*QP evaluation

This repository contains the configurations, fixed inputs, implementations, and recorded outputs used for the map-construction and path-planning experiments in the Polygonal Proximity Map study.

## Scope

- **Experiment 1** evaluates independent-random and scrambled-Halton PPM construction and a multi-resolution grid baseline in four synthetic environments.
- **Experiment 2** compares A* QP with PPM with sampling-based planner: BIT* and grid-based planners: Hybrid-A* and Theta* in the same synthetic environments.
- **Experiment 3** applies the Experiment 2 planner implementations to the real `ku_outdoor` environment. Although it uses separate data and configurations, the planner source is shared with Experiment 2.

## Environments

The four synthetic environments are:

- `01_sparse`
- `02_dense`
- `03_maze_wide`
- `04_maze_narrow`

Experiment 1 result directories use `independent_random` and `halton_random`.
The corresponding internal sampler values remain `random` and `halton`, respectively.

The real environment is `ku_outdoor`,
which is the outdoor area around the Department of Engineering, Kobe University, Japan.
The environmental data was collected by our mobile robot Omnia4.

## Setup

All experiments were conducted on Ubuntu 22.04.

1. Clone the repository with its PPM submodule:

```bash
git clone --recurse-submodules https://github.com/RickNon/ppm_astarqp_evaluation.git
cd ppm_astarqp_evaluation
git submodule update --init --recursive
```

2. Install the Experiment 1 Python dependencies:

```bash
python -m pip install -r experiments/exp1/map_generation/requirements.txt
```

3. Install the additional PPM planner dependencies for Experiment 2 or 3:

```bash
python -m pip install -r experiments/exp2/ppm_planning/requirements.txt
python -m pip install ompl
```

In addition, the Nav2 planners require a separate ROS 2/Nav2 environment. See the [ROS 2 and Nav2 planning instructions](experiments/exp2/README.md#ros-2-and-nav2-planning).

## Timing policy

Planner-query time excludes input loading, shared PPM preprocessing, Nav2 startup and costmap initialization, metric calculation, file output, and plotting.
For A* QP, the measured time includes polygon-adjacency graph construction, A* search, and QP.
For BIT*, it includes OMPL setup, planning, solution simplification, and final path validation, while construction of the PPM validity checker is excluded.
For Hybrid-A*, the stored time is the selected successful request time rather than the wall time of the complete yaw search.

## External software and provenance

- PPM construction uses the `Polygonal-Proximity-Map` git submodule.
- The fixed synthetic Nav2 occupancy grids are generated by the multi-resolution map pipeline from the first depth-wide quadtree frontier that reaches each configured coverage target.
